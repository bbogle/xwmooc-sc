--- layout: base root: ../.. title: 워크샵 점검표(Checklist) — 관리자(admin) ---

관리자(admin)는 소프트웨어 카펜트리 (가상) 본부에 있는 직원으로 강사를 섭외하고, 등록과 평가 준비 및 기타 작업을 도와줌으로써 워크샵 개최를 지원한다.

사전 준비

워크샵 주최자가 워크샵 개최를 결정하기 전에, 소프트웨어 카펜트리 관리자는 주최측을 도와서 워크샵이 언제 어디에서 실시되어야 하는지, 예상 비용을 조언하고, 기타 안내사항을 제공한다.

초기 준비

  • 강사를 섭외한다. 워크샵이 공개적으로 발표되기 전에 완료되어야 한다.
  • 워크샵을 위해서 Github 웹 페이지를 생성하거나 총괄 강사에게 웹페이지 생성을 요청한다.
  • 소프트웨어 카펜트리 site 저장소 config/workshop_urls.yml 파일에 GitHub 저장소 URL을 추가한다.
  • 필요하다면, Eventbrite 이벤트를 설정한다.
    • Eventbrite 이벤트에 주최측 관리자 권한을 추가한다.
    • 내부 데이터베이스에 Eventbrite 이벤트 ID를 추가한다.
  • 워크샵을 발표한다.
    • 소프트웨어 카펜트리 트위터 피드
    • 토론 목록.
  • 소프트웨어 카펜트리 개발 지원을 후원할 능력과 의향이 있는지 개최 기관과 협의한다.
  • 강사의 출장 경비 지원이 가능한지 확인한다.

워크샵 시작 전

  • 주최측, 강사, 조교를 위한 메일링 리스트(mailing list)를 준비한다.
  • 만약 가능다하면, 강사가 출장 경비를 청구하는 방법을 확실히 한다.
  • 만약 필요하다면, 주최측이 조교를 섭외할 수 있도록 돕는다.
  • 사전 워크샵 평가 설문지를 준비하고 학습참가자에게 주최측이 배포하도록 요청한다.
  • 사후 워크샵 평가 설문지를 준비하고 학습참가자에게 워크샵 말미에 배포한다.

워크샵 일주일 전

  • Greg이 강사에게 행운을 기원하는 전자우편을 보내도록 상기시킨다.
  • 만약 취소가 발생한다면, 대기 목록에 있는 다른 학습참여자에게 참여기회를 제공한다.
  • 모든 사항이 원만히 진행되도록 확실히 한다.

워크샵 실시 중

  • 주최측과 강사는 워크샵이 어떻게 진행되고 있는지 살펴보기 위해서 첫째날 전자우편으로 점검한다.

워크샵 실시 후

  • 컨퍼런스 콜(conference call)을 준비해서 Greg이 강사에게서 보고를 받게 한다.
  • 소프트웨어 카펜트리 데이터베이스에 학습참여자와 조교의 이름과 전자우편 주소를 기록한다.
  • 소프트웨어 카펜트리 데이터베이스에 워크샵에 관한 세부사항(날짜, 장소, 강사, 조교 등)이 올바르게 기록되었는지 확실히 한다.
  • 사후 워크샵 평가 설문 결과를 취합한다.
  • 강사가 스스로 강사 사후 평가 설문지를 작성했는지 확실히 한다.
  • 학습 참여자가 메일링 리스트(mailing lists)에 참여하도록 초대한다.
  • 강사, 주최측, 조교가 적절한 배지(badges)를 수여받았는지 확실히 한다.
  • 금번 워크샵에 자리가 없어 참여하지 못하고 대기 명단에 이름을 올린 사람이 있다면, 다른 워크샵을 실시할 때 연락을 취할 명단에 추가한다.
  • 워크샵 실시 2개월 뒤에 장기 사후 워크샵 평가지를 송부하도록 상기시킨다.
